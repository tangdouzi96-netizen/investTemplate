# 投资报告深度分析 Skill

## 核心原则

投资报告必须全面、有据可查。不得以控制篇幅为由压缩内容。数据提取工作可交给低级模型，最终投资判断必须由高级模型完成。

本 skill 存在的原因：模型普遍倾向过度摘要——只写数据 + 估值 + 结论，跳过管理层、护城河、竞争对手、表外风险、极端情景和监控清单。这是失败的报告。

---

## 第一步：检查项目规则

如果当前项目有自己的规则文件（`AGENTS.md`、`docs/ai-rules/`、`readPDF.md`、校验脚本），必须优先遵守。

`investTemplate` 项目必读文件：

- `AGENTS.md`
- `docs/ai-rules/01-报告生成SOP.md`
- `docs/ai-rules/02-数据溯源门禁.md`
- `readPDF.md`
- `scripts/check_report_ready.py`

---

## 模型分工

在不损失质量的前提下节省成本：

| 模型等级 | 适用任务 | 不适用任务 |
|----------|----------|------------|
| 低级模型 | PDF 定位页码、财务报表提取、溯源表草稿、校验脚本修复 | 最终投资结论 |
| 中级模型 | 基于已通过门禁的溯源表生成结构化报告草稿 | 未经核实的数据声明、最终评级 |
| 高级模型 | 最终报告、管理层/护城河/竞对/估值判断、一致性复核 | 纯机械提取（除非必要） |

---

## 硬性执行流程

1. 读取项目必读规则文件
2. 确认年报 PDF 文件存在
3. 只从原始 PDF 提取数据，不使用第三方网站数据
4. 创建 `数据溯源/{公司}_{代码}_数据溯源表.md`
5. 每个核心数字必须标注：来源年份、PDF 页码、单位、口径置信标记
6. 运行 `python scripts/check_report_ready.py {代码}`（脚本存在时）
7. 门禁未通过则停止，不得写报告
8. 报告只能基于已通过门禁的溯源表，以及明确标注来源的市场数据撰写
9. 声称完成前先做自检

---

## 必须有溯源的核心数据

- 营收、毛利、经营利润、净利润、归母净利润
- 现金及现金等价物、存款、金融资产、总资产、总负债、股东权益
- 有息负债（银行借款、债券、租赁负债需分别列示，不得混淆）
- 经营现金流、资本开支、自由现金流、股息、股本总数
- 存货、应收款、应付款、合同负债（视行业相关性）
- 市场数据：股价、市值、汇率、股东事件（标注为外部 C 级来源）

---

## 完整深度报告最低模块清单

新标的完整分析报告不得省略以下任何模块：

1. 数据质量声明（来源表、页码、计算口径、交叉验证）
2. 地缘政治与红线筛查
3. 投资风格分类
4. 负债与财务安全
5. 利润质量
6. 成长历史与周期阶段
7. 竞争对手与商业模式对比
8. 管理层能力评估
9. 公司治理与激励机制
10. 护城河持久性
11. 动态现金流与资本开支分析
12. 估值（含同行参照）
13. 情景分析与买入区间
14. 极端下行压力测试
15. 股东回报分析
16. 表外风险与或有负债
17. 决策流程与仓位建议
18. 持续监控清单

---

## 报告质量规则

- 报告篇幅以足够深入为准，**长度本身不是问题，缺乏深度才是问题**
- 每个评级必须附有原因，包含表格、计算过程和文字判断
- 不得写"来自年报"而不附页码
- 不得将存款、理财产品与现金等价物混淆
- 不得将银行借款与 IFRS 租赁负债混淆
- 不得将财经网站数据作为核心财务数据的 S 级来源
- 数据缺失时写"无法验证"或"待查"，不得编造

## 各模型层推荐提示语

### 低级模型：仅做数据溯源

```text
只做数据溯源，不写投资结论。
读取项目规则、readPDF.md 和数据溯源门禁。
年报在 <folder>。
提取 2021-2025 年核心财务数据，生成 数据溯源/<公司>_<代码>_数据溯源表.md。
每个核心数字必须标 PDF 页码、单位和口径。
完成后运行 python scripts/check_report_ready.py <代码>。
脚本不通过就停止，不要写报告。
```

### 中级模型：生成结构化草稿

```text
基于已通过门禁的数据溯源表，按完整深度报告 18 个模块生成草稿。
禁止省略管理层、护城河、竞对、表外风险、极端情景、监控清单。
找不到的数据标"待查"，不要编造。
不要给最终投资评级，只给草案判断。
```

### 高级模型：终稿审核与重写

```text
作为高级投资分析审稿人，审查并重写最终版。
重点检查：
1. 核心数据是否与溯源表一致；
2. 现金、负债、租赁负债、CAPEX、FCF 口径是否正确；
3. 管理层、护城河、竞对、估值、风险是否有深度；
4. 结论、买点、仓位、风险是否前后一致；
5. 是否有幻觉或未标来源的数据。
请直接重写为最终版，而不是只给修改建议。
```

## 常见失败模式

| 失败行为 | 正确做法 |
|----------|----------|
| "控制篇幅" | 错误。完整报告必须全面，不得压缩。 |
| 未建溯源表就写报告 | 停止，先建溯源表。 |
| 用旧报告数字作为唯一来源 | 替换为 PDF 原文数据。 |
| 将租赁负债归入银行借款 | 分开列示，明确口径。 |
| 将存款/理财混入现金等价物 | 分别列示现金等价物、存款、交易性金融资产。 |
| 对缺失数据自信填写 | 标注缺失或停止。 |
| 最终评级忽视风险章节 | 结论、估值、风险、买点必须保持一致。 |

## 完成前自检清单

声称报告完成前，逐项确认：

- [ ] 已读取项目必读规则文件
- [ ] 数据来自原始年报 PDF
- [ ] 数据溯源表已创建
- [ ] 门禁脚本已通过，或已明确说明阻塞原因
- [ ] 18 个模块全部存在
- [ ] 核心计算数据与溯源表一致
- [ ] 最终结论含评级、买入区间、仓位建议、风险点、监控触发条件
- [ ] 已列出新增/修改的文件路径
- [ ] 未执行 git commit/push（除非用户明确要求）
