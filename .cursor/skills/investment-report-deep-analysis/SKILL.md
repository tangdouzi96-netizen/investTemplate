# 投资报告深度分析 Skill

## 核心原则

投资报告必须全面、有据可查。不得以控制篇幅为由压缩内容。数据提取工作可交给低级模型，最终投资判断必须由高级模型完成。

本 skill 存在的原因：模型普遍倾向过度摘要——只写数据 + 估值 + 结论，跳过管理层、护城河、竞争对手、表外风险、极端情景和监控清单。这是失败的报告。

## 模型分工（成本优先：DeepSeek 包揽低+中两段）

> **用户策略**：尽可能让 DeepSeek 完成数据溯源 + 报告草稿，高级模型只做验收与补深度，不整篇重写。

| 阶段 | 执行者 | 任务 | 禁止 |
|------|--------|------|------|
| **第一段** | DeepSeek（低级） | PDF 定位、财报提取、溯源表；运行 `check_report_ready.py` | 写报告、写投资结论、用 WebSearch 填核心财务数 |
| **门禁** | 脚本 | `check_report_ready.py` 返回 1 → 打回（V5.5.16：表头契约/勾稽/核心行✅/—占位/A-CF） | 跳过门禁写报告 |
| **第二段** | DeepSeek（中级） | 按 `template/00-草稿骨架.md` 填空写草稿；数字只引溯源表编号 | 凭记忆/搜索产生新数字；留空；造数 |
| **终审** | 高级模型 | ①数字与溯源表对账 ②补全所有 `待查` ③结论/买点/风险一致性 | 通读重写整篇（除非草稿严重不合格） |

**待查铁律**：第二段搞不定的判断一律写 `待查（高级模型补）` 或 `草案判断，待高级确认`，禁止编造。

**两段硬切**：同一会话也须先交溯源表 + 门禁通过，再开报告骨架；不得边提取边写结论。

## 硬性执行流程

1. 读取项目必读规则文件
2. 确认年报 PDF 文件存在
3. 只从原始 PDF 提取数据，不使用第三方网站数据
4. 创建 `数据溯源/{公司}_{代码}_数据溯源表.md`
5. 每个核心数字必须标注：来源年份、PDF 页码、单位、口径置信标记
6. 运行 `python scripts/check_report_ready.py {代码}`（脚本存在时）
7. 门禁未通过则停止，不得写报告
8. 报告只能基于已通过门禁的溯源表，以及明确标注来源的市场数据撰写
9. 声称完成前先做自检

---

## 必须有溯源的核心数据

- 营收、毛利、经营利润、净利润、归母净利润
- 现金及现金等价物、存款、金融资产、总资产、总负债、股东权益
- 有息负债（银行借款、债券、租赁负债需分别列示，不得混淆）
- 经营现金流、资本开支、自由现金流、股息、股本总数
- 存货、应收款、应付款、合同负债（视行业相关性）
- 市场数据：股价、市值、汇率、股东事件（标注为外部 C 级来源）

---

## 完整深度报告最低模块清单

新标的完整分析报告不得省略以下任何模块：

1. 数据质量声明（来源表、页码、计算口径、交叉验证）
2. 地缘政治与红线筛查
3. 投资风格分类
4. 负债与财务安全
5. 利润质量
6. 成长历史与周期阶段
7. 竞争对手与商业模式对比
8. 管理层能力评估
9. 公司治理与激励机制
10. 护城河持久性
11. 动态现金流与资本开支分析
12. 估值（含同行参照）
13. 情景分析与买入区间
14. 极端下行压力测试
15. 股东回报分析
16. 表外风险与或有负债
17. 决策流程与仓位建议
18. 持续监控清单

---

## 报告质量规则

- 报告篇幅以足够深入为准，**长度本身不是问题，缺乏深度才是问题**
- 每个评级必须附有原因，包含表格、计算过程和文字判断
- 不得写"来自年报"而不附页码
- 不得将存款、理财产品与现金等价物混淆
- 不得将银行借款与 IFRS 租赁负债混淆
- 不得将财经网站数据作为核心财务数据的 S 级来源
- 数据缺失时写"无法验证"或"待查"，不得编造

## 各模型层推荐提示语

### DeepSeek 第一段：仅做数据溯源

```text
【第一段·禁止写报告】
读取 AGENTS.md、readPDF.md、docs/ai-rules/02-数据溯源门禁.md。
年报在 <folder>。
提取 2021-2025 年核心财务数据，生成 数据溯源/<公司>_<代码>_数据溯源表.md。
每个核心数字必须标 PDF 页码、单位和口径；分红/股息必须用年报人民币原文，禁止 WebSearch 港币。
受限资金：必须查附注「所有权或使用权受到限制的资产」。
完成后运行 python scripts/check_report_ready.py <代码>，贴返回码。
返回 1 则停止，不要进入第二段。
```

### DeepSeek 第二段：按骨架填草稿

```text
【第二段·禁止产生新数字】
确认门禁已通过（返回 0 或 2）。
复制 template/00-草稿骨架.md 为报告草稿，逐节填写。
所有数字只引用溯源表编号（如 A07）；判断不足写「待查（高级模型补）」。
6 个易漏小节必填：公司概览、受限资金、两职合一、护城河侵蚀、买入价公式、期望回报 vs 无风险利率。
买入价必须写：公式 → 代入 → 结果。
垄断/平台企业必须答「什么能侵蚀这个护城河？」
不要给最终投资评级，只给草案判断。
写完再运行 python scripts/check_report_ready.py <代码>，处理警告项。
```

### 高级模型：验收终稿（非整篇重写）

```text
【终审·聚焦清单】
输入：已通过门禁的溯源表 + DeepSeek 草稿。
任务：
1. 抽查报告每个核心数字 ↔ 溯源表编号是否一致（防造假）；
2. 全文搜索「待查」，逐项补成最终判断（深度类由你完成）；
3. 结论、买点、仓位、风险是否前后一致；
4. 仅对不合格章节重写，合格章节保留。
输出最终版 07-分析输出/{公司}_{代码}_投资分析报告.md。
```

## 常见失败模式

| 失败行为 | 正确做法 |
|----------|----------|
| "控制篇幅" | 错误。完整报告必须全面，不得压缩。 |
| 未建溯源表就写报告 | 停止，先建溯源表。 |
| 用旧报告数字作为唯一来源 | 替换为 PDF 原文数据。 |
| 将租赁负债归入银行借款 | 分开列示，明确口径。 |
| 将存款/理财混入现金等价物 | 分别列示现金等价物、存款、交易性金融资产。 |
| 对缺失数据自信填写 | 标注缺失或停止。 |
| 最终评级忽视风险章节 | 结论、估值、风险、买点必须保持一致。 |

## 完成前自检清单

### DeepSeek 第二段（交高级模型前）

- [ ] 门禁脚本已运行并贴返回码
- [ ] 使用 `template/00-草稿骨架.md`，18 模块 + 6 易漏小节未删
- [ ] 无凭记忆/搜索补的核心财务数字
- [ ] 买入价含公式+代入+结果
- [ ] 搞不定的已标 `待查`，无造数

### 高级模型终审后

- [ ] 已读取项目必读规则文件（含 `06-全面交叉校验清单.md`）
- [ ] 数据来自原始年报 PDF，与溯源表一致
- [ ] 终稿含「合理性交叉校验（第4层）」：股息率/支付率/累计分红 vs 市场或年报
- [ ] 所有 `待查` 已处理或明确保留原因
- [ ] 最终结论含评级、买入区间、仓位建议、风险点、监控触发条件
- [ ] 已列出新增/修改的文件路径
- [ ] 未执行 git commit/push（除非用户明确要求）
