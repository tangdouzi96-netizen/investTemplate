## 一、溯源表结构标准

### A 区：直接提取项（PDF → 溯源表 → 验证）

| 编号 | 科目 | {Y-4} | {Y-3} | {Y-2} | {Y-1} | {Y} | PDF({Y}) | WS |
|------|------|-------|-------|-------|-------|-----|:------:|:--:|
| A01 | 营业收入 | x亿 | x亿 | x亿 | x亿 | x亿 | pXX | ✅ |
| A02 | 归母净利润 | x亿 | x亿 | x亿 | x亿 | x亿 | pXX | ✅ |

每个数字必须：
1. 附 PDF 页码（不标页码 = 置信度降级）
2. 附 WS 验证标记（空白 = 未验证 = 阻断）

WS 列图例见 [readPDF.md §1](#) 或 [readPDF.md §4](#)。

### B 区：衍生计算项（溯源表计算 → 报告引用）

| 编号 | 指标 | 公式 | 代入值 | 结果 |
|------|------|------|--------|------|
| B01 | 有息负债 | 短期借款+一年内到期+长期借款+应付债券 | A22+A24+A25+A26 | xx亿 |
| B02 | 净现金 | 货币资金-有息负债 | A14-B01 | xx亿 |
| B03 | FCF | 经营现金流-CAPEX | A30-A31 | xx亿 |

公式代入值必须引用 A 区或 C 区编号，确保可追溯。

### C 区：外部数据项（非年报来源）

| 编号 | 数据项 | 数值 | 来源 | 时点 |
|------|--------|------|------|------|
| C01 | 股价 | ¥xx | 东方财富/雪球 | YYYY-MM-DD |
| C02 | 市值 | ~xx亿 | 股价×总股本 | YYYY-MM-DD |
| C03 | 行业数据 | xx | 统计局/行业协会 | YYYY |

### 报告引用格式

报告中引用溯源表数据时使用编号标注：`营业收入 257.31 亿（溯-A01）`。

## 二、行业特有科目扩展

A 区核心科目覆盖所有公司共用部分。不同行业追加：

| 行业 | 追加科目 |
|------|---------|
| **白酒** | 中高档酒营收、中高档酒毛利率、其他酒营收、其他酒毛利率 |
| **煤炭** | 原煤产量、可采储量、吨煤成本、长协价占比 |
| **电力** | 装机容量、利用小时数、上网电价、度电成本 |
| **消费/零售** | 门店数、同店增速、客单价、会员数 |
| **中药/保健品** | 零售终端数（按地区）、自产/贸易收入占比、产品注册进度 |
| **银行** | 净息差、不良率、拨备覆盖率、核心一级资本充足率 |
| **REITs** | 可供分配金额、出租率、租金单价、加权平均租期 |

## 三、报告中"溯源表没有"的数据怎么处理

| 类别 | 处理方式 | 示例 |
|------|---------|------|
| **能进 C 区的** | 收入溯源表 C 区，标注来源 | 行业数据、同业对比 |
| **进不了的** | 紧挨数字写假设前提 | "假设 2026 年利润再降 15%→约 92 亿" |
| **禁止出现** | 无来源、凭感觉的数字 | — |

## 四、一致性自检清单

- [ ] A 区每个数字是否标注了 PDF 页码
- [ ] WS 列是否无空白（空白=未验证→阻断）
- [ ] BS 平衡验算是否通过（资产=负债+权益，容差<0.1%）
- [ ] 流动资产合计是否可反推（各子项相加≈合计，容差<1%）
- [ ] CF 平衡验算是否通过（经营+投资+筹资+汇率=净增加额）
- [ ] B 区公式代入值是否全部引用 A 区或 C 区编号
- [ ] 报告中所有数字是否可追溯到 A/B/C 区编号
- [ ] 同一指标在报告不同位置数值是否一致
- [ ] 时期数 vs 时点数是否正确（BS=期末，PL/CF=全年）
- [ ] 合并报表 vs 母公司报表是否明确区分
- [ ] 单位换算是否正确（元→亿÷100,000,000、万元→亿÷10,000、千港元→亿港元÷100,000）
- [ ] 是否存在追溯调整，如有是否标注双口径（readPDF.md 红线 8）
- [ ] 空值科目是否 WebSearch 确认（readPDF.md 红线 6）
- [ ] 同比/环比百分比是否重新计算确认

## 五、用户抽查方法

```
1. 打开年报 PDF，随机选 3 个页码
2. 在溯源表中找到对应页码的数据行
3. 对比 PDF 原文值 vs 溯源表值 → 必须完全一致
4. 再选 2 个 B 区衍生计算项，代入 A 区值手算验证
5. 结论：
   5 个全对 → 信任其余，报告可用
   1 个有误 → 要求 AI 回溯全流程，找出根因
```
