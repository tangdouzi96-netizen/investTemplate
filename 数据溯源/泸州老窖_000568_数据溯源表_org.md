## 十、方法论备注

### 关键风险点（本次验证中暴露的）

1. **合并报表 vs 母公司报表混淆**：OCF 在合并 CF(p82)=71.23亿，母公司 CF(p83)=45.06亿（2025年）。**每年两个值，一律使用合并报表**。
2. **PDF 跨行截断**：CAPEX 科目名被截断为"购建固定资产、无形资产和其他长\n期资产支付的现金"，简单 grep 找不到。必须用"购建固定"搜索。
3. **CF 补充资料标签混淆**：补充资料中的"资产减值准备"与 PL 中的"资产减值损失"不是同一概念。
4. **科目标签张冠李戴**：将"固定资产折旧 6.80亿"标为"资产减值损失"是致命错误。
5. **单位换算错误**：元→亿 ÷ 100,000,000，必须二次确认。

### 验证流程

```
每次生成/更新报告后，强制执行：
1. 关键科目（营收/净利/OCF/CAPEX/折旧/减值）PDF 页码定位 + 原文拍照
2. WebSearch 同一科目 + 同一报告期交叉验证
3. 合并报表 vs 母公司报表明确标注（禁止混用）
4. 单位换算二次确认（元→亿÷100,000,000）
5. 写入数据溯源表，逐项勾选
```
