# 05 · DeepSeek 两段式通用提示词（对话 0～3）

> **用途**：低成本模型（DeepSeek 等）包揽 **数据溯源 + 报告草稿**；高级模型只做 **终审验收**。  
> **用法**：每次分析新标的，复制对应「对话块」，把 `{占位符}` 替换成实际值；**对话 0→1→2 分三次发送**（不要合并成一条，避免边溯源边写报告）。  
> **上位规则**：`AGENTS.md`、`docs/ai-rules/01-报告生成SOP.md`、`template/00-草稿骨架.md`

---

## 占位符说明（替换后再发）

| 占位符 | 示例 | 说明 |
|--------|------|------|
| `{公司名称}` | 中国平安 | 中文全称 |
| `{股票代码}` | 601318 | A 股 6 位或港股 5 位（报告/溯源表文件名用） |
| `{市场}` | A股 / 港股 | 决定估值标准（A>4%股息 / 港>6%） |
| `{年报目录}` | 07-分析输出/中国平安/年报/ | 相对项目根目录，放 2021-2025 年报 PDF（子目录名按实际调整） |
| `{工作目录}` | D:\Project\investTemplate | 本机项目绝对路径 |
| `{行业备注}` | 保险/金融：FCF倍数不适用，偿付能力+内含价值；银行/保险豁免6关部分项 | 非金融可写「无」或删除该段 |
| `{护城河侵蚀示例}` | 利率下行、监管、互联网保险、代理人渠道 | 按行业改，勿照搬航空 NDC/OTA |
| `{买入估值公式}` | 保险：买入价=目标P/EV×每股内含价值；工商：买入价=(FCF倍数×FCF−净现金)÷股数÷汇率 | 按行业选 |

**产出文件（固定命名）**：
- 溯源表：`数据溯源/{公司名称}_{股票代码}_数据溯源表.md`
- 报告草稿/终稿：`07-分析输出/{公司名称}_{股票代码}_投资分析报告.md`

> **何时用**：`07-分析输出/index.md` 尚无该公司，或用户要求先筛再深挖。  
> **何时跳过**：已分析标的、仅更新年报。

```text
你是投资研究助手，只做新标的初筛，不写报告、不写溯源表。

标的：{公司名称}
股票代码：{股票代码}（{市场}）

必读（按顺序）：

2. {工作目录}/docs/ai-rules/01-报告生成SOP.md 中的「6关定量筛选」

任务：
- 若有 PDF，核心筛选数据只从 PDF 取；无 PDF 则标明「缺年报，筛选数据待补」
```

```text
【第一段·禁止写报告】

标的：{公司名称}
股票代码：{股票代码}
市场：{市场}
工作目录：{工作目录}
年报 PDF 目录：{年报目录}（相对工作目录）

必读文件（必须先读再动手）：
- AGENTS.md
- readPDF.md
- docs/ai-rules/02-数据溯源门禁.md

【V5.5.16 准确性硬约束 · 必读】

表头契约（写在溯源表最上方元信息，门禁会阻断）：
- **币种**：主币种（如人民币）；若早年港币须写明换算公式与汇率页码，或拆 HKD/RMB 子表
- **单位**：主单位（如亿元）；写明 PDF 列示单位→表内换算（例：千元÷100000=亿元）
- 禁止同一 5 年列混用不同币种/单位裸比

1. 先建表结构：A-PL / A-BS / A-CF / A-DIV / A-NOTE / B区
2. 对每个核心科目，必须分别打开 2021AR、2022AR、2023AR、2024AR、2025AR 各自 PDF 提取
   - 禁止只读最新年报综合报表某一页（如 2024AR p122）然后其他年份留 —
   - 禁止用「业务摘要 p6-7」代替正式合并报表页码（摘要仅作交叉验证）
3. BS 可拆成 A-BS-1（近年明细）+ A-BS-2（2021-2023 关键项），但全局须覆盖 2021-2025

1. 只做数据溯源表，禁止写投资分析报告、禁止写最终投资评级
2. 每个核心数字必须来自年报 PDF 原文，标注 p+页码、单位、口径（合并/母公司、万元/亿元）
3. 分红/股息/现金等禁止用 WebSearch、财经网站或港币行情冒充 S 级数据；港股标的也须用年报披露币种原文
4. 必须查附注「所有权或使用权受到限制的资产」（受限资金）；无受限也要在 A-NOTE 写明
5. {行业备注}
6. 缺数据标 ⚠️ 或 🔴，**禁止用 — / 空白 表示「未提取」**
7. 禁止用 📝 代替缺数（📝 仅表示「已从 PDF 提取且第三方无法精确验证」）
8. 5 年表内核心科目须 2021–2025 全填；必备科目全局年份覆盖（门禁会阻断）
9. 必须含 A-CF 章节（经营现金流 + CAPEX 行），禁止编造

WS 列与验证汇总表（见 `06-全面交叉校验清单.md` §3，门禁 V5.5.18）：

**A 类科目**（营收、归母、经营CF、CAPEX、每股股息**合计**、分红总额）须在 **WebSearch 验证汇总** 表按年标记：
- **2025**：尽量 ✅ 或 📄（公告级）
- **2022–2024**：✅ 或 🔗（次年比较列差值=0），**禁止 📝**
- **2021**：📝 即可（**不要求 WebSearch**），禁止 ⚠️/空白

**B 类组**（现金、总负债、资产总计）：组内至少一行 ✅/🔗/📄。
**C 类明细**（费用、BS 子项）：主表允许 📝 + 勾稽。
**🔗** = 次年年报比较列确认；禁止用 WebSearch 填 A 区数值本身。

勾稽（必填，缺通过标记 → 门禁阻断）：
- 自检块须含 **BS 平衡验算 … ✅**（最新年：资产=负债+权益）
- 自检块须含 **CF 平衡验算 … ✅**（经营+投资+筹资+汇率≈现金净增）
- 建议附 PL 勾稽（收入−成本−费用≈营业利润）

禁止事项：
- 禁止 WebSearch 填 A 区核心财务数
- 缺 PDF 或某年未发布 → 该行标 ⚠️ 并说明原因，不得编造

必须写入溯源表末尾的「自检块」：

| 检查项 | 结果 |
| A 区 — 占位数量 | 0 |
| A-CF 含经营CF+CAPEX | 是/否 |
| 2021-2025 营收/归母/经营CF/CAPEX/现金/总负债 各年有值 | 是（注明分散在哪张表） |
| BS 平衡验算（最新年） | 公式+结果 |
| CF 平衡验算（最新年） | 公式+结果 |
| 受限资金（A-NOTE） | 数值+页码或「无受限」 |
| 币种/单位一致性 | 通过 + 换算公式 |
| 核心组双校验（现金/总负债/资产） | 至少一行 ✅ |
| WebSearch 验证汇总 | 已含；2025✅ / 2022–2024🔗可 / 2021📝可 |
| BS 平衡验算 | 公式+✅ |
| CF 平衡验算 | 公式+✅ |

产出（必须实际写入文件）：
- 数据溯源/{公司名称}_{股票代码}_数据溯源表.md

完成后必须运行并贴完整终端输出：
cd {工作目录}
python scripts/check_report_ready.py {股票代码}

回复必须包含：
- 完整终端输出（含返回码）
- 若返回码为 1：逐条解释阻断项 + 修正计划，**禁止说「可以进入第二段」**
- 若返回码为 2：列出全部 ⚠️ 行，说明是否能在本对话内补完；**仍不要写报告**
- 若返回码为 0：粘贴自检块 + 列出每个核心科目「2021→2025 来源页码一览」

仅当返回码为 0 且 A 区无 — 占位时，用户才可进入对话 2。
```

```text
1. 抽查报告核心数字 ↔ 溯源表编号是否一致（防 WebSearch/记忆造假）

- 回复中列出：对账结果、已补待查项、门禁返回码、未解决项
```

## 常见错误（勿合并对话）

| 错误做法 | 正确做法 |
