# 04 · 模拟组合 / 决策追踪数据规则

> 本文件为长规范。涉及模拟组合、每日监控、AI 决策记录时必须先读取本文件。
> 上位规则见根目录 `AGENTS.md`。

---

## 单一真相源原则

唯一权威数据源：`08-决策追踪/simulation_trades.csv`

所有持仓数据以 trades.csv 为准：

- 成本价 = 成交价格
- 持股数 = 成交股数净额
- 现金 = 初始资金 − 买入金额 + 卖出金额
- code = 5 位字符串（保留前导 0）

禁止：在代码中硬编码成本价/持股数/现金；多处维护同一份数据；直接改 `simulation_state.json` 而不更新 trades.csv。

---

## 强制验证

每次运行模拟组合脚本前：

```bash
python scripts/validate_simulation_data.py
```

验证 trades.csv 结构、state 与 trades 一致性、持仓字段完整、现金计算。失败返回 1 则停止，人工修复后再继续。该脚本只读不改。

---

## 数据修复 SOP

```
1. 停止主脚本
2. 运行 validate_simulation_data.py
3. 定位问题
4. trades.csv 错 → 改 trades.csv；state.json 错 → 删 state.json 让脚本重建
5. 重新验证
6. 恢复运行
```

## AI 决策记录

明确买入/卖出/加减仓建议（含目标价）时，必须记录到 `08-决策追踪/AI决策记录.md`：

```markdown
| 日期 | 标的 | 代码 | 操作 | 建议价格 | 当前价格 | 收益率 | 决策理由 | 状态 |
```

"持有不新增/观望/无具体价位"不需记录。状态：🟢正确 / 🔴错误 / 🟡待验证 / ⚪已平仓 / ❓待确认。

## VHSI 市场情绪联动

给仓位建议前查 `08-决策追踪/vhsi_monitoring.json`（过旧>3天则跑 `python scripts/fetch_vhsi.py`）。

| VHSI | 调整 |
|------|------|
| <22 | 正常 |
| 22-27 | 仅核心仓位 |
| 27-32 | 暂停新建仓 |
| 32-40 | 🟠🔴标的建议减仓 |
| >40 | 仅被错杀龙头卫星试探 |

禁止：VHSI>32 时建议重仓买入不符合"净现金>市值"标准的标的。
