# 02 · 数据溯源门禁

> 本文件为长规范。提取/核对财报数据、生成报告前必须先读取本文件与 `readPDF.md`。
> 上位规则见根目录 `AGENTS.md`。

---

## 核心原则

年报 PDF 和公司公告优先于一切第三方数据源。宁可留白待核实，不得用第三方数据补齐核心财务数字。

---

## 必须溯源的核心数据

现金及现金等价物、定期/受限存款、有息负债（借款明细加总）、营业收入、净利润与归母净利润、经营现金流、CAPEX（购建固定资产）、股本、分红、回购。

---

## 门禁条件（进入报告写作前必须全部满足）

运行 `python scripts/check_report_ready.py {代码}` 返回 0 或 2，且：

- 核心财务数据均来自 PDF/公告原文。
- 每个核心数字标注页码（如 `p80`）或附注编号。
- 合并报表与母公司报表已区分。
- 单位已确认（万元/亿元/千元）。
- 资产负债表、现金流量表关键项完成交叉校验。
- 未解决 🔴（差值≥5%）数量 = 0。

脚本判定规则（V5.5.16+）：

| 条件 | 结果 |
|------|------|
| 🔴 未解决 | 阻断（返回 1） |
| **表头契约缺币种或缺单位声明** | **阻断** |
| **缺 BS/CF 平衡验算通过标记（✅）** | **阻断** |
| **核心摘要行 WS 为 📝/⚠️**（营收/归母/经营CF/CAPEX/股息等） | **阻断**（若验证汇总表已按年分层通过，主表 A 类可 📝） |
| **验证汇总表 A 类 2022–2025 分层违规**（2025 禁 📝；2022–2024 禁 📝） | **阻断** |
| **核心组无 ✅**（现金/总负债/资产总计组内至少一行） | **阻断** |
| 缺 WebSearch 验证汇总区块 | 警告 |
| 2025 年验证汇总仅 🔗 无 ✅ | 警告 |
| A 区核心科目缺页码 | 阻断 |
| 核心行用 WebSearch/搜索且无页码 | 阻断 |
| 缺 A-CF 或缺经营 CF / CAPEX 行 | 阻断 |
| 5 年表内核心行用 `—` 占位 | 阻断 |
| 必备科目缺 2021–2025 任一年 | 阻断（可分表） |
| 股本 | 至少一年有 PDF 来源即可 |
| ⚠️ 超过 3 条 | 警告（返回 2） |
| 核心科目疑用港币 | 警告 |
| 报告缺 6 易漏小节或买入区间无公式 | 警告 |

**禁止用 `—` 表示「未提取」**；**禁止核心摘要行用 📝 代替双校验**（2021 年例外见下）。

### WS 年份分层（A 类 · 验证汇总表）

| 年份 | 合格 | 不合格 |
|------|------|--------|
| **2025** | ✅、📄（🔗 可过，建议 ✅） | 📝、⚠️、空白 |
| **2022–2024** | ✅、🔗、📄 | 📝、⚠️、空白 |
| **2021** | 📝、✅、🔗、📄 | ⚠️、空白 |

完整四层校验见 `docs/ai-rules/06-全面交叉校验清单.md`。

> 低成本模型分工见 `template/00-草稿骨架.md` 与 `docs/ai-rules/01-报告生成SOP.md`「模型分工」节。

## 数据置信度标注

| 置信度 | 条件 | 标注示例 |
|--------|------|----------|
| S级 | 年报原文附注，已核查 | `S级（2025年报P56附注22）` |
| A级 | 业绩公告/官网，与年报一致 | `A级（业绩公告，与年报一致）` |
| B级 | 基于年报计算，含估算 | `B级（含估算成分）` |
| C级 | 第三方平台，待验证 | `C级（东方财富，待年报验证）` |
| D级 | 无法验证 | `D级（禁止用于投资决策）` |

---

## 失败处理

任一门禁失败时停止，并输出：缺失数据项、已查来源、无法继续原因、需要用户提供/确认的材料。

---

## 常见数据错误类型

| 类型 | 案例 | 预防 |
|------|------|------|
| 时点错误 | 用中报代替年报 | 强制年报，除非未发布 |
| 科目混淆 | 把总负债当有息负债 | 从借款附注明细加总 |
| 单位错误 | 万元当亿元 | 确认报表单位 |
| 估算代替 | 净现金用估算 | 从附注精确计算 |
| 置信度虚标 | 实际B级标S级 | 严格按标注规范 |
| PDF优先缺失 | 已有PDF却用第三方 | 先查 07-分析输出/ 下PDF |

> 完整错误案例见 `docs/ai-rules/99-错误案例库.md`。
> PDF 读取规范见根目录 `readPDF.md`（五条红线 + 页码标注 + 合并/母公司区分）。
