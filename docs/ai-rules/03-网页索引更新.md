# 03 · 网页索引更新

> 本文件为长规范。**默认非强制**——仅当用户提到"网站 / VitePress / 侧边栏 / 首页展示 / 索引"时执行。
> 上位规则见根目录 `AGENTS.md`。

---

## 背景

本项目网页用 VitePress 把 Markdown 渲染成网站，没有手写 HTML 报告页（仅 `public/` 下有少量独立 HTML 仪表盘）。
报告文件存在 ≠ 网站能找到。报告需登记到三处"入口"才能在网站导航中出现。

---

## 三处索引

| # | 文件 | 作用 |
|---|------|------|
| 1 | `07-分析输出/index.md` | 分析输出目录页（按行业分类的表格） |
| 2 | `index.md` | 首页"📈 最新分析报告"表格 |
| 3 | `.vitepress/config.mjs` | 左侧侧边栏链接 |

---

## 更新方法

**目录页** `07-分析输出/index.md` —— 在对应行业表格加一行：

```markdown
| [公司名](./公司名_代码_投资分析报告.md) | 代码 | 流派 | 核心结论 | 日期 |
```

**首页** `index.md` —— 在"最新分析报告"表格加一行（带评级图标 🐢🍊/🟡/⚠️/🔴）：

```markdown
| 🟡 **公司名_代码** | 评级 | 核心结论 | [查看](/07-分析输出/公司名_代码_投资分析报告) |
```

**侧边栏** `.vitepress/config.mjs` —— 在 `07-分析输出` 的 `items` 内加：

```javascript
{ text: '公司名_代码_投资分析报告', link: '/07-分析输出/公司名_代码_投资分析报告' },
```

---

## 校验

```bash
python scripts/check_index_sync.py
```

返回 0 = 三处均已同步；返回 1 = 打印缺失清单。执行索引更新后必须重跑确认返回 0。
