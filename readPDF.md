# 年报 PDF 读取规范

> **适用**：A 股 / 港股年报 PDF 数据提取  
> **工具**：Python PyMuPDF (`fitz`) — 唯一可靠方案  
> **禁止**：bash `pdftotext` 操作中文路径、Windows Store Python  

---

## 〇、前置铁律

> **任何标的在开始写投资分析报告之前，必须先完成数据溯源表。**

### 流程顺序

```
快速筛选 → 数据溯源表 →（✅率达标）→ 投资分析报告
```

具体操作按 [§1 标准化操作 SOP](#一标准化操作-sop四步走) 执行，达标标准见该节末尾。

### 溯源表结构

```
数据溯源表
├── A 区：直接提取项（PDF 原文值 + 页码 + WS 验证标记）
├── B 区：衍生计算项（公式 + 代入值 + 结果）
└── C 区：外部数据项（股价/市值/行业数据，标注来源时间）
```

每个从年报提取的数字，必须经过：**PDF 读取 → 标注页码 → WebSearch 核对 → 标记 WS**（详见[§4 WebSearch 交叉验证流程](#四websearch-交叉验证流程)）。

## 一、标准化操作 SOP（四步走）

> 新标的做数据溯源时，按以下四步执行。高危陷阱详见[§7 高危科目清单](#七高危科目清单)。

### 步骤 1：dump → 定位报表页

```
搜"审计报告"（A股）或"INDEPENDENT AUDITOR"（港股）
  → 往后翻 4-6 页找到三大报表首页
  → 确认页首含"合并"而非"母公司"（A股）/ "CONSOLIDATED"（港股）
  → 记录 PL / BS / CF 起始页码
```

A 股定位词：`合并资产负债表` | `合并利润表` | `合并现金流量表`（备选：`一、营业总收入`）
港股定位词：`CONSOLIDATED STATEMENT OF PROFIT OR LOSS` | `... FINANCIAL POSITION` | `... CASH FLOWS`
港股备选：搜 `Cash generated from operations` 且首行不含 `NOTES`

### 步骤 2：批提取 → 每数标页码和单位

```
从步骤 1 确定的起始页，读取 2-3 页全文
  → 逐行匹配科目名，取紧随其后的会计数值（跳过附注编号等小数字）
  → 每条标注：PDF 页码 + 原始单位（元 / 万元 / 千港元）
  → 强制执行平衡验算（不通过→排查重来）：
      BS: 资产总计 = 负债合计 + 所有者权益合计
      CF: 经营CF + 投资CF + 筹资CF + 汇率 = 现金净增加额
      PL: 收入 - 成本 - 税金 - 三费 - 减值 ≈ 营业利润
```

### 步骤 3：交叉验证

```
用次年年报的比较列逐一核对（如 2021 年 OCF 在 2022 年 CF 中应重现）
  → 差值 = 0 → 通过
  → 差值 ≠ 0 → 排查是否为追溯重述（详见 §3 红线 8）
```

覆盖范围：OCF、CAPEX、InvestCF 等 CF 科目；PL/BS 核心科目可用摘要页交叉确认。

### 步骤 4：并行 WebSearch

```
6-8 个搜索词同时发送
  → 核心科目（收入/利润/总资产/分红）：每条必搜
  → 明细科目（费用/BS 子项）：港股中小市值通常搜不到，标注 📝
  → WS 列按实际验证层级填入，不虚假标注
```

搜索关键词模板和判定标准详见 [§4 WebSearch 交叉验证流程](#四websearch-交叉验证流程)。

WS 列图例：

| 标记 | 含义 | 何时使用 |
|:--:|------|------|
| ✅ | WebSearch 独立确认 | 财经网站有该科目精确数据，差值 ≤ 1% |
| 🔗 | 相邻年报交叉验证 | CF 科目经次年比较列确认一致 |
| 📄 | 10 年摘要页 | 港股特有，核心 PL/BS 经审计摘要页确认 |
| 📝 | 仅 PDF 提取 | 明细科目（费用/BS 子项），已过平衡验算 |
| 🟡 | 差值 < 5% | 警告，标注差异原因 |
| 🔴 | 差值 ≥ 5% | 待排查（排查定因后→📝，排查不清→阻塞） |
| ⚠️ | 待补充 | 未做任何验证 |

### 达标标准

| 指标 | 标准 | 不达标后果 |
|------|------|------|
| ✅ / 🔗 / 📄 / 📝 合计 | **≥ 95%** | 禁止进入报告阶段，回溯补齐 |
| ⚠️ 数 | **≤ 3 条**，且均为 P3 级（经营数据类：产量/销量/售价） | 同上 |
| 未解决的 🔴 | **必须为 0** | 存在→立即回溯排查，排查后仍无法解决→阻塞报告 |

### A 类验证汇总表 · 按年份分层（V5.5.18+）

须在溯源表末尾维护 **「WebSearch 验证汇总」** 表；A 类科目按年标记，规则见 `docs/ai-rules/06-全面交叉校验清单.md` §3.2：

| 年份 | 合格标记 | 说明 |
|------|----------|------|
| **2025** | ✅、📄（🔗 可过，建议 ✅） | 决策主依据 |
| **2022–2024** | ✅、🔗、📄 | 可不 WebSearch，次年比较列 🔗 即可 |
| **2021** | 📝、✅、🔗、📄 | **不要求 WebSearch** |

主表 A 区 WS 列：核心摘要行禁止 ⚠️/空白；若验证汇总表已按上表通过，主表允许 📝。

## 二、页码范围速查

中国年报结构高度标准化，每个数据类别对应固定页码范围：

| 数据类别 | 页码范围 | 搜索关键词示例 |
|----------|:---:|------|
| 主要财务指标摘要 | p5-12 | `营业收入`、`归母净利润`、`加权平均净资产收益率`、`基本每股收益` |
| **合并**资产负债表 | p70-78 | `货币资金`、`存货`、`资产总计`、`负债合计`、`长期借款` |
| **合并**利润表 | p78-82 | `营业收入`、`营业成本`、`销售费用`、`财务费用` |
| **合并**现金流量表 | p82-86 | `经营活动产生的现金流量净额`、`购建固定资产` |
| 分部/分产品报告 | p12-40 | `中高档酒类`、`分产品`、`分地区` |
| CF 补充资料（间接法） | p100+（附注尾部） | `固定资产折旧`、`无形资产摊销`、`资产减值准备` |
| 分红/股本信息 | p2-6 | `派`、`总股本`、`利润分配` |

### 定位技巧：三级 fallback 策略

```
一级关键词 → 标准年报关键词
  ↓ 失败
二级关键词 → 备选关键词
  ↓ 失败
三级全扫描 → 全文档逐页扫描科目名
```

#### 一级关键词（标准化年报）

```python
import fitz
doc = fitz.open("年报.pdf")

bs_start = pl_start = cf_start = None

for pg in range(doc.page_count):
    text = doc[pg].get_text("text")
    if "合并资产负债表" in text and "编制单位" in text and "母公司" not in text[:300]:
        bs_start = pg
    if "合并利润表" in text and "编制单位" in text and "母公司" not in text[:300]:
        pl_start = pg
    if "合并现金流量表" in text and "编制单位" in text and "母公司" not in text[:300]:
        cf_start = pg
```

#### 二级关键词（非标准年报，备选定位）

一级关键词未命中时启用。已知非标准案例：

| 报表 | 公司案例 | 一级关键词(失败) | 二级关键词(成功) |
|------|---------|------|------|
| PL | 陕西煤业 2022-2025 | `合并利润表` | `一、营业总收入`（且不含"母公司"） |
| CF | 陕西煤业 2021-2025 | `合并现金流量表` | `一、经营活动产生的现金流量`（且不含"母公司"） |
| BS | 部分港股 | `合并资产负债表` | `綜合財務狀況表` / `综合财务状况表` |

```python
# 二级 fallback
if pl_start is None:
    for pg in range(doc.page_count):
        text = doc[pg].get_text("text")
        if "母公司" in text[:300]:
            continue
        if "一、营业总收入" in text[:500]:
            pl_start = pg
            break

if cf_start is None:
    for pg in range(doc.page_count):
        text = doc[pg].get_text("text")
        if "母公司" in text[:300]:
            continue
        if "一、经营活动产生的现金流量" in text:
            cf_start = pg
            break
```

#### 三级全扫描（终极 fallback）

二级也未命中时，对全文档逐页搜索目标科目关键词，人工确认页面是否为合并且非母公司。

```python
def full_scan_keyword(doc, keyword):
    """全文档扫描，返回所有匹配页码及上下文"""
    results = []
    for pg in range(doc.page_count):
        text = doc[pg].get_text("text")
        if keyword in text:
            idx = text.find(keyword)
            ctx = text[max(0,idx-100):min(len(text),idx+300)]
            results.append({'page': pg+1, 'context': ctx[:200]})
    return results

# 示例：定位不到CF页时，全文档搜 CAPEX 关键词
matches = full_scan_keyword(doc, "购建固定资产")
for m in matches:
    print(f"p{m['page']}: ...{m['context']}...")
```

> ⚠️ 三级全扫描返回的结果**必须人工逐条确认**，判断是否位于合并报表页（检查页面是否含"母公司"、是否含"合并"或报表特征关键词）。

## 三、八条红线（违反任一条 → 数据无效）

### 红线 1：合并 vs 母公司必须区分

年报同时包含**合并报表**和**母公司报表**，两张表页码紧挨，数值差异巨大。

| 科目 | 合并报表 | 母公司报表 | 差异 |
|------|---------|-----------|------|
| 泸州老窖 OCF | **71.23亿** (p82) | 45.06亿 (p83) | 36.7% |
| 泸州老窖 总负债 | **149.01亿** (p76) | 109.42亿 (p78) | 26.6% |

**规则**：除母公司持股比例分析外，**一律使用合并报表**。提取前确认页面上有"合并"二字。

### 红线 2：科目名跨行截断

PDF 文本提取时，长科目名可能被换行符截断：

```
PDF 原文：     购建固定资产、无形资产和其他长\n期资产支付的现金
简单 grep：    找不到 ← 漏掉
正确做法：     搜"购建固定" → 看上下文确认完整科目
```

**规则**：搜索关键词用科目名的**前 4-6 个字**，而非全称。搜到后看上下文 3-5 行确认是目标科目。

### 红线 3：单位换算二次确认

年报数字单位通常是**元**，但部分公司摘要页用**万元**。报告统一用**亿元**。换算公式：

```
亿元 = 元 ÷ 100,000,000
亿元 = 万元 ÷ 10,000
```

**常见错误**：小数点移位（19.33 亿 → 1.93 亿）、万/亿混淆、**同一份年报不同页面单位不同**（摘要页用万元、报表正文用元，陕西煤业案例）。

**识别方法**：进入新页面先检查表格标题行是否有"单位：万元"或"单位：元"。摘要页（主要会计数据）大概率用万元，报表正文大概率用元。

**规则**：提取后立即标注 `{原始值}_{单位}`，换算后复核数量级是否合理（对比该科目历史值，突变 > 30% 需回溯原文）。

### 红线 4：每个数字必须标注 PDF 页码

**规则**：写入报告前，每个数字必须附 `(pXX)` 页码。不标页码 = 无法追溯 = 置信度自动降为 B 级。

```
✅ 经营现金流 71.23亿（p82 合并CF）
❌ 经营现金流 71.23亿（来源：年报）
```

### 红线 5：时点确认

| 报表类型 | 时点格式 | 示例 |
|----------|---------|------|
| 资产负债表 | `2025_末` | 时点数，不能与利润表直接对比 |
| 利润表 | `2025_全年` | 时期数 |
| 现金流量表 | `2025_全年` | 时期数 |

**规则**：每取一个数字，立即标注 `{年份}_{报告期}`。BS 科目不能与 PL/CF 科目混用时点。

### 红线 6：空值科目必须 WebSearch 确认

PDF 表格中某科目存在但无数值（空值 = 0）时，脚本极易从相邻行抓取数字填入，造成"空值变有值"。

| 案例 | 科目 | 脚本误填 | 实际值 | 误填来源 |
|------|------|:--:|:--:|------|
| 陕西煤业 2025 | 应付债券 | 138.75亿 | **0** | 长期借款的上期值 |

**规则**：
- 脚本提取值为 0 或空 → **必须** WebSearch 确认该科目确实为 0
- 脚本提取值恰好等于相邻科目的值（如上期值、子项值）→ **必须** WebSearch 确认不存在串行
- 确认后在溯源表中标注 `0¹⁰` 并注明 WebSearch 验证来源

### 红线 7：提取后必须全文 dump + 平衡等式验算

脚本输出 JSON → 直接填表是最危险的路径。脚本正确率约 70%，30% 的科目存在串行、小数点错位、跨行截断等问题。

**规则**：
1. 脚本提取完成后，**必须**将 BS/PL/CF 页全文 dump 为 txt 文件
2. **必须**人工逐行通读 dump 文本，逐一核对脚本输出的每个值
3. BS **必须**验算：`流动资产合计 ≈ 各流动资产相加`、`资产总计 = 负债合计 + 所有者权益合计`（容差 < 0.1%）
4. PL **必须**验算：`营业收入 - 营业成本 - 税金及附加 - 三费 - 减值 = 营业利润`（容差 < 1%，因有未提取的科目）
5. CF **必须**验算：`经营CF + 投资CF + 筹资CF + 汇率影响 = 现金净增加额`（容差 < 0.1%）

**反例**：陕西煤业应收账款 5.93 亿 → 用"流动资产合计 - 其他各项"反推得 59.25 亿，立即暴露 10 倍错误。

### 红线 8：追溯调整必须保留双口径 + 标注调整方法

中国会计准则下，同一控制下企业合并需追溯重述前期数据。次年报告的上期列可能与当年报告自身数据不同。

| 案例 | 科目 | 2023年报自身 | 2024年报追溯调整 | 差异 |
|------|------|:--:|:--:|:--:|
| 陕西煤业 | 资产总计 | 1,939亿 | 2,360亿 | +421亿(陕煤电力并表) |

**规则**：
1. 溯源表 A 区主值以**各年年报自身 PDF** 为准（保证可追溯）
2. 若次年报告存在追溯调整，在主表中增加 `(追溯后:xxx)` 标注
3. 附录中列出双口径对比表 + 调整原因
4. **若调整方法为追溯重述法（同一控制下合并）→ 标记 ⚠️ 预警**，因该方法会显著改变历史财务数据，直接影响趋势分析和历史对比
5. 分析报告中跨年趋势对比时，使用追溯调整后的可比口径

**常见调整方法判断**：

| 调整方法 | 典型场景 | 影响程度 | 预警 |
|---------|---------|:--:|:--:|
| 追溯重述法 | 同一控制下企业合并 | 大（资产/负债/权益全变） | ⚠️ |
| 会计政策变更 | 新准则实施 | 中（特定科目） | 🟡 |
| 会计估计变更 | 折旧年限调整等 | 小（仅影响当期及以后） | — |
| 前期差错更正 | 更正错报 | 视情况 | ⚠️ |

## 四、WebSearch 交叉验证流程

> PDF 提取和 WebSearch 核对应在**同一个流程步骤中完成**，禁止分开执行（分开 = 必然跳步）。

### 每条 A 区数据的标准处理链

```
PDF 读取值 → 标注页码 → WebSearch 搜索 → 数值比对 → 标记 ✅/🟡/🔴
```

这五个步骤必须**逐条 A 区科目连续完成**，不得抽样跳过。每完成一条，立即在溯源表中填入验证标记。

### 判定标准

| 判定 | 标准 | 含义 |
|:--:|------|------|
| ✅ | PDF 原文值 = WebSearch 值，差值 ≤ 1% | 通过，可直接使用 |
| 🟡 | 差值 < 5% | 警告，标注差异原因（如四舍五入、口径差异） |
| 🔴 | 差值 ≥ 5% | **需要排查**，不是最终判定。按[🔴解决流程](#溯源表完成标准)排查定因后→📝，排查不清→阻塞 |
| ⚠️ | WebSearch 未找到精确匹配 | 降级置信度，优先用 PDF 原文值，标注"待补充验证" |

### WebSearch 搜索关键词模板

| 数据类型 | 搜索模板 |
|------|------|
| 营收/利润 | `{公司名} {代码} {年份}年报 营业收入 归母净利润` |
| BS 科目 | `{公司名} {代码} {年份} {科目名} 资产负债表` |
| CF 科目 | `{公司名} {代码} {年份} CAPEX 购建固定资产 现金流量表` |
| 空值确认 | `{公司名} {代码} {科目名} 2025 2024 资产负债表` |
| 分红 | `{公司名} {代码} {年份} 分红 每股股利 总股本` |
| 经营数据 | `{公司名} {代码} {年份} {产量/销量/售价} 年报` |

### 财经网站追溯重述值的判别

财经网站（东方财富、Investing.com、新浪财经等）的历史数据往往使用**最新年报追溯调整后**的重述值，而非各年年报原始值。

| 判别信号 | 说明 |
|------|------|
| PDF 原文值 ≠ 网站值，但 = 次年报告的上期列 | 网站使用了追溯重述值 |
| 多个网站显示同一数值 ≠ PDF 原文 | 网站之间共享数据源，不代表正确 |
| 差额恰好是某重大收购的资产规模 | 确认是同一控制下合并的追溯重述 |

**规则**：
1. 以 PDF 原文值为准（可追溯到具体页码）
2. 如果网站值 ≠ PDF 值，**先排查是否为追溯重述差异**，不要直接判 PDF 读错
3. 如确认为追溯重述差异 → 溯源表保留 PDF 原值，附录中标注双口径（参见红线 8）
4. 如排除追溯重述，网站值 ≠ PDF 值且差值 ≥ 5% → 🔴，走解决流程

### 逐条验证 ≠ 抽样验证

> ⚠️ 陕西煤业 V1.1 只对营收/净利/资产负债率做了 WebSearch 验证，BS 明细和 CF 明细全跳过了。这直接导致应付债券=0 被误填为 138.75 亿、应收账款 10 倍错误未被发现。

**必须逐条验证的科目**：

| 优先级 | 科目类型 | 风险 |
|:--:|------|------|
| 🔴 最高 | 空值科目（应付债券、短期借款某年为 0） | 脚本极易串行填入相邻值 |
| 🔴 最高 | 余额变动 > 50% 的科目 | 可能是单位/口径错误 |
| 🟡 高 | 应付/应收类科目 | 小数点错位高发 |
| 🟡 高 | 投资活动/筹资活动 CF | 跨行截断 → 取到母公司值 |

## 五、科目缺失处理规则

溯源表中某个单元格为空时，必须按以下规则填写。关键词：**`0` 永远不能直接填，必须带脚注。**

| 情况 | 写法 | 是否需要 WebSearch | 示例 |
|------|:--:|:--:|------|
| 公司根本无此业务 | `—` | 否 | 2021-2023 年陕西煤业无电力业务→A34 写 `—` |
| 公司有此业务但当年无发生额 | `—` 或 `0`(注) | 视情况 | 某年无处置子公司收益 |
| **公司有此科目但余额 = 0** | **`0` + 脚注** | **是（强制）** | 应付债券已偿还→`0¹⁰`（WebSearch 确认） |
| 数据未提取到/待补充 | `⚠️` | 是 | CAPEX 跨行截断未读到 |

> **核心区别**：`—` 表示"不存在"，无需验证；`0` 表示"存在但余额为零"，**必须** WebSearch 确认，因为空值科目是脚本串行的最高发区域。

**脚注格式**：`> ¹⁰ 应付债券 2021-2025 年在 BS 页显示为空值(=0)，WebSearch 确认公司无存续债券。`

## 六、提取验证口令

### 单科目自问

每读完一个数字，自问五句：

```
1. 合并还是母公司？（不是"合并"就要找合并的）
2. 第几页？（没记页码现在回去记）
3. 单位是元还是万元？（元 → 亿 ÷ 1亿，万元 → 亿 ÷ 1万）
4. 这个"合计"是子项还是总项？（流动负债合计 ≠ 负债合计）
5. 空值还是真有值？（科目出现但无数字 → WebSearch 确认是 0 还是漏读）
```

五句全过，数据才能写入报告。

### 全表交叉验算（每张表读完强制执行）

```
BS 验算：
  资产总计 = 负债合计 + 所有者权益合计  → 容差 < 0.1%
  流动资产合计 ≈ 各流动资产科目相加      → 容差 < 1%
  非流动资产合计 ≈ 各非流动资产科目相加  → 容差 < 1%

PL 验算：
  营业收入 - 营业成本 - 税金 - 三费 - 减值 ≈ 营业利润 → 容差 < 1%

CF 验算：
  经营CF + 投资CF + 筹资CF + 汇率影响 = 现金净增加额 → 容差 < 0.1%
  期末现金 = 期初现金 + 现金净增加额                  → 容差 < 0.1%
```

验算不通过 → 立即回溯原始 dump 文本，逐行排查串行/漏读/单位错误。

## 七、高危科目清单

以下科目在 PDF 表格中排列密集，最易串行/误读：

| 高危科目 | 所在报表 | 常见错误 | 检测方法 |
|----------|---------|---------|---------|
| CAPEX（购建固定资产…） | 合并 CF | 科目名跨行截断、与理财投资混淆 | 搜索"购建固定" |
| 应付债券 | 合并 BS | **空值串行**：科目存在但余额=0，脚本抓取相邻行（长期借款上期值/租赁负债）填入 | WebSearch 确认 |
| 一年内到期非流动负债 | 合并 BS | 与"一年内到期非流动资产"混淆；空值时被跳过 | 流动资产合计验算 |
| 应收账款 | 合并 BS | **小数点错位**：元→亿 ÷100,000,000 时漏除 | 流动资产合计反推 |
| 短期借款 | 合并 BS | 某年余额=0 时，脚本填入上期值或相邻值 | WebSearch 确认 |
| 资产总计 / 负债合计 | 合并 BS | **高危匹配陷阱**：前面紧挨"流动资产合计""流动负债合计""非流动资产合计""非流动负债合计"，模糊搜索极易匹配到子项，导致数值差 10-35 倍。必须精确匹配整行，排除前缀干扰（如 `line.strip() == '负债合计'` 或排除含"流动""非流动"的行） | 平衡等式验算 |
| 固定资产折旧 | CF 补充资料 | 与"资产减值准备"标签混淆 | — |
| 无形资产摊销 | CF 补充资料 | 被误标为"折旧" | — |
| 资产减值损失 | 合并 PL | 与 CF 补充资料"资产减值准备"混淆 | — |
| 利息费用 | 合并 PL vs CF 补充 | 两个口径数值不同 | — |
| 投资活动现金流 | 合并 CF | 科目名跨行截断 → 找不到 → 误取母公司CF值 | 确认页面有"合并"或排除"母公司" |
| 归母净资产 | 合并 BS | 科目名跨行截断（"归属于母公司所有者权益合计"通常跨 2-3 行） | 搜索"归属于母公司" |

## 八、Python 提取标准代码

```python
import fitz, re, json, sys
sys.stdout.reconfigure(encoding='utf-8')

doc = fitz.open("年报.pdf")

def search_pages(keyword, start, end, ctx=3):
    """在指定页码范围搜索关键词，返回页码+上下文"""
    results = []
    for pg in range(start, min(end, doc.page_count)):
        text = doc[pg].get_text("text")
        lines = text.split('\n')
        for li, line in enumerate(lines):
            if keyword in line:
                s = max(0, li - ctx)
                e = min(len(lines), li + ctx + 1)
                results.append({
                    'page': pg + 1,
                    'matched': line.strip()[:150],
                    'context': [l.strip()[:120] for l in lines[s:e]],
                })
    return results

def to_yi(raw_str, unit='yuan'):
    """元/万元 → 亿，返回 float。unit='yuan' 或 'wan'"""
    v = float(raw_str.replace(',', ''))
    if unit == 'wan':
        return round(v / 10_000, 2)      # 万元 → 亿
    return round(v / 100_000_000, 2)     # 元 → 亿

def dump_pages_to_file(doc, pages, out_path):
    """将指定页面全文 dump 到 txt 文件（红线7强制要求）"""
    with open(out_path, 'w', encoding='utf-8') as f:
        for pg_num in pages:
            text = doc[pg_num - 1].get_text("text")
            f.write(f"\n{'='*60}\n")
            f.write(f"📄 第{pg_num}页\n{'='*60}\n")
            f.write(text)
    print(f"✅ 全文dump → {out_path}")

def verify_bs_balance(assets, liabilities, equity):
    """BS平衡验算：资产 = 负债 + 所有者权益"""
    diff = abs(assets - liabilities - equity)
    pct = diff / assets * 100 if assets > 0 else 999
    if pct > 0.1:
        raise AssertionError(
            f"❌ BS不平衡！资产{assets:.2f} ≠ 负债{liabilities:.2f} + 权益{equity:.2f}，"
            f"差异{pct:.2f}%。请回溯 dump 文本排查。"
        )
    print(f"✅ BS平衡通过（差异{pct:.3f}%）")

def verify_cf_balance(operating_cf, investing_cf, financing_cf, fx_impact, net_cash_change):
    """CF平衡验算：经营+投资+筹资+汇率 = 现金净增加额"""
    total = operating_cf + investing_cf + financing_cf + fx_impact
    diff = abs(total - net_cash_change)
    pct = diff / abs(net_cash_change) * 100 if net_cash_change != 0 else diff
    if pct > 0.1:
        raise AssertionError(
            f"❌ CF不平衡！合计{total:.2f} ≠ 净增加额{net_cash_change:.2f}，"
            f"差异{pct:.2f}%。请回溯 dump 文本排查。"
        )
    print(f"✅ CF平衡通过（差异{pct:.3f}%）")

# ── 使用示例 ──
# 1. 提取前先 dump 全文
dump_pages_to_file(doc, [80, 81, 82], "bs_pages.txt")  # BS页
dump_pages_to_file(doc, [88, 89, 90], "cf_pages.txt")  # CF页

# 2. 提取后立即验算（不通过则抛异常，阻断后续流程）
# verify_bs_balance(assets=2362.69, liabilities=988.62, equity=1374.07)

doc.close()
```
